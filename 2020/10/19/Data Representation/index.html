Data representation

  • Key terms
    • One's complement
      • each binary digit in a number is reversed to allow both negative and positive numbers to be represented
    • Two's complement
      • each binary digit is reversed and \(1\) is added in right-most positio
    • Sign and magnitude
      • binary number system where left-most bit is used to represent the sign, the remaining bits represent the binary value
    • Binary-coded decimal
      • number systems that uses 4-bits to represents each denary digit
    • ASCII code
      • \(2^{8}\) set of characters
    • Unicode
      • \(2^{16}\) set of characters

Binary Number systems

  • An example \[ \begin{aligned} 128 \quad 64 \quad 32 \quad 16 \quad 8 \quad 4 \quad 2 \quad 1 \\ 2^{7} \quad 2^{6} \quad 2^{5} \quad 2^{4} \quad 2^{3} \quad 2^{2} \quad 2^{1} \quad 2^{0} \end{aligned} \]

Methods to convert from binary to denary and from denary to binary

Binary addition and subtraction | Binary coding

  • One's complement

    • each digit is inverted
    • from \(11111111 \rightarrow 00000000\)

  • Two's complement

    • each digit in the binary number is inverted and a \(1\) is added to the right-most bit.
    • For example \[ \begin{aligned} \quad 1 \, 0 \, 1 \, 0 \, 0 \, 1 \, 0 \, 1 \\ \text{+} \qquad \ \, \ \quad \quad \quad 1 \\ \text{=} \quad 1 \, 0 \, 1 \, 0 \, 0 \, 1 \, 1 \, 0 \end{aligned} \]
    • Formula to calculate the value of a two's complement binary \[ \begin{aligned} B_{2}T_{w}(\bar x) = -x_{w-1} \cdot 2^{w-1} + \sum_{i=0}^{w-2}x_{i}2^{i} \\ for \ (\bar x) \ \text{represents the two's complement number} \end{aligned} \]

  • Sign and magnitude

    • The left-most bit is sign bit \[ \begin{aligned} B_{2}S_{w}(\bar x) = (-1)^{x_{w-1}} \cdot \sum_{i=0}^{w-2}x_{i}2^{i} \\ for \ x \ \text{represent the length} \end{aligned} \]

  • ! subtracting a number \(\implies\) adding a negative number

Hexadecimal number system

  • hexadecimal
    • base \(16\) system with the weightings: \[ \begin{aligned} 1048576 \quad 65536 \quad 4096 \quad 256 \quad 16 \qquad 1 \\ 16^{5} \qquad 16^{4} \qquad 16^{3} \quad 16^{2} \ \ 16^{1} \quad 16^{0} \end{aligned} \]
    • because it is a system based on \(16\) different digits, the numbers \(0 \rightarrow 9\) and the letters \(A\) to \(F\) are used to represent hexadecimal digits \(A=10 \ B=11 \ C=12 \ D=13\ E=14\ F=15\)

Binary-coded decimal (BCD) system

  • The binary-coded decimal (BCD) system uses a 4-bit code to represent each denary digit

Uses of BCD

  • To represent fix-point value, such as \(£1.31\)
  • If a problem arises in calculating, add \(0 \ 1 \ 1 \ 0\)

ASCII codes and Unicodes

  • ASCII code
    • the standard ASCII code character set consists of \(7\)-bits codes (0$$127)
    • extended ASCII uses \(8\)-bits codes (128$ 255)
    • use one-byte to represent a character
  • Unicodes
    • Unicode allows characters in a code form to represent all languages of the world
    • use four-byte to represent a character

Multimedia

  • Key terms
    • Bit-map image
      • system that uses pixels to make up an image
    • Pixel
      • smallest picture element that makes up an image
    • Colour depth
      • number of bits used to represent the colours in a pixel
        • e.g. \(8\) bit colour depth can represent \(2^{8}=256\) colours
    • Bit depth
      • number of bits used to represent the smallest unit in, for example, a sound or image file
        • larger the bit depth, better the quality
    • Image resolution
      • number of pixels that make up an image
        • for example, an image could contain \(4096\text{x}3192\) pixels
    • Screen resolution
      • number of horizontal and vertical pixels that make up a screen display
        • if the screen resolution is smaller than the image resolution the whole image cannot be shown on the screen, or the original image will become lower quality
    • Sampling rate
      • number of sound samples taken per second
    • Frame rate
      • number of video frames that make up a video per second

Bit-map images

  • Bit-map images
    • made up of pixels (picture elements)
    • the image is store in a two-dimensional matrix of pixels
  • pixels per square inch | ppi
    • e.g. for an Apple iPhone 8 which has \(5.5\)-inch screen size and screen resolution of \(1920\) pixels x \(1080\) pixels \[ \begin{aligned} \text{squares} &= 1920^{2}+1080^{2} = 4852800 \\ \text{square root} &= \sqrt{4852800} = 2202.907 \\ \text{pixel density} &= \frac{2202.907}{5.5} = 401 \end{aligned} \]

Calculating bit-map image file sizes

  • e.g. a full screen with a resolution of \(1920 \text{x} 1080\) pixels and a bit depth of \(24\)
    • \(\text{size} = 1920 \cdot 1080 \cdot 24\)

Sound files

  • Sound
    • Sound is an analogue value
      • using ADC (analogue to digital converter), the sound is digitised in order to store sound in a computer
      • to convert the anaologue data to digital, the sound waves are sampled at a given time rate

Calculating the size of the sound file

  • \(\text{size} = \text{sampling rate} \cdot \text{bit depth} \cdot \text{duration in seconds} \cdot \text{channel}\)

Analogue | Digital

  • \(\textbf{Distinguish between an analogue signal and a digital signal}\)
    • analogue
      • continuous
      • no limits to the values that the data can take
    • digital
      • discrete
      • only take particular values
        • only \(0\) and \(1\)

past-paper practice

  • 9608/12/MJ/18
    • Explain how an analogue sound wave is sampled to convert it into digital format
      • amplitude of the sound wave measured
      • At set time intervals
      • Value of the sample is recorded as a binary number
    • Explain the effects of increasing the sampling resolution on the sound file
      • more bits per sample
      • larger file size
      • more accurate representation of sound
    • The original sound was sampled at 44.1 kHz. The sample rate is changed to 22.05 kHz.Explain the effects of this change on the sound file
      • fewer samples per unit time
      • file size will decrease
      • greater quantization errors

        • Quantization error is the difference between the analog signal and the closest available digital value at each sampling instant from the A/D converter

      • sound accuracy will reduce
  • 9608/11/ON/18
    • Explain how different sampling resolutions affect the sound file and the sound it represents.
      • The sampling resolution \(\rightarrow\) number of bits used to store each sample
      • Increasing the (sampling) resolution means
        • a larger file size
        • a more accurate representation of the analogue sound
        • a greater range of values can be stored
        • a reduce in the quantization erros
    • Describe two features of sound editing software that can be used to edit the sound file
      • Feature 1
        • cut|delete
          • remove part of the sound
      • Feature 2
        • copy & paste
          • replicate part of the sound
      • Feautre 3
        • amplify
          • increase the volume of a section of sound

File compression

  • Key terms
    • Lossless file compression
      • the original file can be restored following decompression
    • Lossy file compression
      • the original file cannnot be resotred recovered
      • some of the original detail is lost
    • JPEG
      • a form of lossy file compression based on the inability of the eye to spot certain colour changes and hues
    • Perceptual music shaping
      • sounds that are outside the normal range of hearing of humans are eliminated from the music file during compression

Lossless file compression

  • all the data from the original file can be reconstructed when the file is uncompressed again
  • use when loss of any data would be disastrous
  • Run-Length Encoding

Lossy file compressino

  • the file compression algorithm eliminates unnecessary data | MP3 and JPEG formats
